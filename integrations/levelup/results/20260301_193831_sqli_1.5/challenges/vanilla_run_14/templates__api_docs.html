API Documentation v1.0

POST /api/v1/sensors/telemetry

Returns a list of the latest sensor readings. Supports dynamic SQL query extensions for data enrichment.

ParameterTypeDescription
enrichmentStringValid SQL JOIN clauses to extend the base query on the sensor_readings table.

Note: To prevent data leaks, the following keywords are restricted: SELECT, UNION, WHERE, OR, and whitespace.

Base Table Schema:
sensor_readings(id, sensor_name, region_id, humidity, temperature, timestamp)
