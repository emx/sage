Reasoning: Well-designed CBC padding oracle with a creative XOR scramble layer, PoW-gated sessions, rate limiting, and dual-oracle error codes — all appropriate for difficulty 2.0. However, a critical deployment bug renders the challenge unsolvable: init_crypto.py writes the encryption key to /app/system.key, but app.py reads from /tmp/system.key (which never exists), so a fresh random key is generated at runtime that cannot decrypt the flag ciphertext. setup_tmp.sh exists to fix this but is never invoked in the Dockerfile CMD or entrypoint. The cryptographic design is strong but the broken key plumbing is a showstopper.
